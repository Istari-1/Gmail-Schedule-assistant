
# Gmail Calendar Bot
Google Apps Script (GAS) と Gemini をベースにした、スマート日程自動化アシスタントです。Gmail を自動的に監視し、AI を活用してメール内のスケジュール情報を解析、指定された Google カレンダーに同期すると同時に、Google Chat へ美しくインタラクティブなカード通知を送信します。

---

## コア機能

- **スマート解析 (Intelligent Parsing)：** Gemini を利用し、メールからイベント名、開始/終了時間、場所、および会議リンクを正確に抽出します。
- **自動カレンダー同期 (Auto Sync)：** 解析に成功したスケジュールを、指定された Google カレンダーに自動登録します（1つのメールから複数のスケジュールを同時に抽出することも可能です）。
- **エラーハンドリングと再試行 (Fault Tolerance)：** 5回の指数バックオフ（Exponential Backoff）再試行メカニズムを内蔵。API で頻発する `503 Service Unavailable` エラーに柔軟に対応します。
- **安全なデータ秘匿化 (Security)：** GAS の「スクリプトプロパティ」で API Key を管理するため、コード内からの機密情報漏洩を防止します。
- **インタラクティブ通知 (Interactive Notification)：** Google Chat の Webhook へカード形式の通知を送信。ボタンをクリックするだけで、予約リンクへのジャンプや手動でのカレンダー追加が可能です。
- **過去スケールのフィルタリング (Past Event Filtering)：** メール内の過去の日程を自動的に識別してスキップし、カレンダーを常に整理された状態に保ちます。

## ファイル構成

### 1. Bot側 (bot) 说明
- `Main.js`：メインロジック。メールの検索、プロセスの制御、およびカレンダー書き込みのスケジュール管理を担当。
- `Gemini.js`：コア AI 処理モジュール。プロンプトエンジニアリングおよび 503 エラーの再試行メカニズムを含む。
- `sendCard.js`：Google Chat 用のインタラクティブなカードオブジェクトの構築と送信を担当。
- `Config.js`：グローバル定数の定義およびスクリプトプロパティの読み込み。
- `sendMessage.js`：Google Chat へのテキストメッセージ送信を担当。

### 2. 勤務/大学用メインメール側 (shadow_forwarder)
- `コード.js`：メールの転送処理ロジック。
- `Config.js`：グローバル定数の定義およびスクリプトプロパティの読み込み。

## クイックスタート

### app scripts および Gemini api 方面

#### 事前準備
- 転送されたメールを処理するための「Bot専用 Gmail アカウント」を1つ作成します。
- Gemini API Key を取得します。
- Google Chat でスペースを作成し、Webhook URL を取得します（カード通知の受信に使用）。
- 同期先の Google カレンダー ID を取得します（メインのカレンダーを使用する場合は `primary` で構いません）。

#### Bot側スクリプトのデプロイ
1. 在 Google Apps Script 创建新项目。**このスクリプトはBot用メールアカウントの環境下に配置してください。**
2. bot プロジェクト内のすべての `.js` ファイルの内容を、それぞれ対応する `.gs` ファイルにコピーします。
3. 左側のメニューから「プロジェクトの設定」 (歯車アイコン) をクリックし、以下の「スクリプトプロパティ」を追加します：
    - `GEMINI_API_KEY`：取得した Gemini API Key
    - `CHAT_WEBHOOK`：Google Chat の Webhook URL
    - `TARGET_CALENDAR_ID`：スケジュールを登録するカレンダー ID（省略時は自動的にメインカレンダーになります）

#### メインメール側スクリプトのデプロイ
1. 在 Google Apps Script 创建新项目。**このスクリプトはメインメールアカウントの環境下に配置してください。**
2. shadow_forwarder プロジェクト内のすべての `.js` ファイルの内容を、それぞれ対応する `.gs` ファイルにコピーします。
3. 左側のメニューから「プロジェクトの設定」 (歯車アイコン) をクリックし、以下の「スクリプトプロパティ」を追加します：
    - `EMAIL_BOT`：先ほど作成した Bot用 Gmail アドレス

#### トリガーの設定 (両方のアカウントで実施)
1. 両方のスクリプトプロジェクトで、左側のメニューから「トリガー」 (時計アイコン) をクリックします。
2. 「トリガーを追加」 をクリックし、実行する関数として `main` を選択、イベントのソースを「時間主導型」に設定します。推奨する頻度は **10分おき** です。

### メインメールの設定

正確なスクリプトの実行とお互いの無限ループを防ぐため、メインの Gmail 側で2つの補完的なフィルタを設定します。また、転送状況をマーク・管理するために「待中転 (転送待ち)」と「已中転 (転送済み)」の2つのラベルを作成します。

#### 1. 内部転送フィルタ
- **一致条件：** `subject:([TRANS]) from:me to:あなたのbotメール@gmail.com`
- **実行する操作：** 「受信トレイをスキップ (アーカイブする)」、「既読にする」
- **無限ループ防止：** スクリプトが `sendTextMessage` などで新しいメールを生成することがあるため、このフィルタによって、Bot宛てかつ `[TRANS]` マークが付いたシステム内部メールが再度スクリプトをトリガーするのを防ぎ、直接アーカイブします。システム用の指令メールがメインの受信トレイを煩わせることはありません。

#### 2. 日程キーワードフィルタおよび自身の送信メールの除外
- **一致条件：** `{試験 テスト プレゼンテーション 月 日 講座 開催 予約 申込 リマインド セミナー ガイダンス 締切 講義 授業 面接 イベント 案内 説明会 オリエンテーション} -from:me`
- **実行する操作：** ラベル「待中転」を適用
- **説明：** 「試験、テスト、プレゼンテーション、予約、面接、説明会」など20以上の日本の大学日程に関連するキーワードを含みます。
- `-from:me`：これは非常に重要な安全スイッチです。自分が送信した返信や下書きを除外することで、スクリプトが自身の送信メールを解析して無限ループに陥るのを防ぎます。

#### 3. ラベル (Label) の作成
- メインメール側で「待中転」および「已中転」という2つのラベルを手動で作成し、スクリプトの転送状態管理に使用します。

---

### bot メールの設定

- **設定不要：** Bot側の Gmail アカウントでの特別なフィルタ設定は必要ありません。すべてスクリプト側で自動処理されます。

# Gmail Calendar Bot
一个基于 Google Apps Script (GAS) 和 Gemini  的智能日程自动化助手。它能自动监控 Gmail 邮件，利用 AI 解析其中的日程信息，并同步到指定的 Google 日历中，同时向 Google Chat 发送美观的交互式卡片。

## 核心功能
1. 智能解析：利用 Gemini 提取邮件中的活动名称、开始/结束时间、地点及会议链接。

2. 自动同步：解析成功的日程将自动写入指定的 Google 日历，支持多场次解析。

3. 容错重试：内置 5 次指数退避重试机制，有效应对 API 频繁出现的 503 Service Unavailable 错误。

4. 安全脱敏：支持通过 GAS 脚本属性管理 API Key，避免敏感信息泄露。

5. 交互通知：向 Google Chat Webhook 发送卡片通知，支持直接点击按钮跳转预约链接或手动加日历。

6. 过期过滤：自动识别并跳过邮件中的过往日程，确保日历整洁。

## 技术栈
语言：Google Apps Script (JavaScript)

模型：Google Gemini 

集成服务：Gmail API, Calendar API, Google Chat Webhook

## 文件说明


1. bot端(bot)说明

    ```Gemini.js```: 核心 AI 处理模块，包含 Prompt 提示词工程及 503 重试机制。

## 快速开始
### app scripts以及Gemini api方面
1. 准备工作

- 创建bot邮箱用于处理转发来的邮件

- 获取 Gemini API Key。

- 创建一个 Google Chat 并获取此对话的 Webhook 地址（用于接收卡片消息）。

- 获取目标日历 ID（若使用主日历，填 primary 即可）。
******
2. 部署bot端脚本
在 Google Apps Script 创建新项目。
**此脚本应布置在使用bot邮箱登陆的Google Apps Scripts下**

- 将```bot```项目中的所有 ```.js``` 文件内容复制到对应的 ```.gs``` 文件中。

- 点击左侧 “项目设置” (齿轮图标)，添加以下 “脚本属性”：

    1. ```GEMINI_API_KEY```: 你的 Gemini API Key

    2. ```CHAT_WEBHOOK```: 你的 Google Chat Webhook URL

    3. ```TARGET_CALENDAR_ID```: 接收日程的日历 ID（可选，默认为主日历）
******
3. 部署工作邮箱端脚本
在 Google Apps Script 创建新项目。
**此脚本应布置在使用工作邮箱登陆的Google Apps Scripts下**
- 将```shadow_forwarder```项目中的所有 ```.js``` 文件内容复制到对应的 ```.gs``` 文件中。

- 点击左侧 “项目设置” (齿轮图标)，添加以下 “脚本属性”：

    1. ```EMAIL_BOT```: 你的 bot邮箱号

******

4. 设置触发器 **两个邮箱下的脚本都要设置**
    点击左侧 “触发器” (闹钟图标)。

    添加触发器：选择 main 函数，部署为“时间驱动”，建议频率为 每 10 分钟一次。



### Gmail工作邮箱设置方面
为了实现精准触发并防止死循环，在 Gmail 工作邮箱侧需要配置了两个互补的过滤器；同时为了标记是否中转设置了标签（label）```待中转```和```已中转```

1. 内部中转过滤
    ```
    匹配条件：
    subject:([TRANS]) from:me to你的bot邮箱@gmail.com
    ```

    >执行操作：跳过收件箱、标记为已读。

    防止死循环：由于脚本会通过 sendTextMessage 等方式产生新邮件，。这个过滤器确保所有发往机器人邮箱、且带有``` [TRANS] ```标记的邮件不会再次触发脚本，直接存入库中。确保这些系统内部的指令邮件不会骚扰你的主收件箱。


******
2. 日程关键词过滤以及排除自己发送的邮件
 
    ```
    匹配条件：
    {試験 テスト プレゼンテーション 月 日 講座 開催 予約 申込 リマインド セミナー ガイダンス 締切 講義 授業 面接 イベント 案内 説明会 オリエンテーション} -from:me
    ```
    >执行操作： 应用标签“待中转”

    包含“考试、测试、演示、讲座、预约、面试、说明会”等 20 多个符合日本大学日程安排邮件的关键词。


    ```-from:me```：这是一个关键的安全开关。它排除了你自己发送的邮件，防止脚本解析你发出去的回复或草稿形成循环。
******
3. 创建```已转发```和```待转发```标签(label)
